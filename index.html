HTML: La estructura de una página web

HTML (HyperText Markup Language) es el lenguaje estándar para crear la estructura de las páginas web. Define los elementos que componen una página, como encabezados, párrafos, imágenes, enlaces, etc. Estos elementos se organizan en una jerarquía que determina la estructura visual de la página.

CSS: Dando estilo a las páginas web

CSS (Cascading Style Sheets) es un lenguaje de diseño que se utiliza para describir la presentación de un documento escrito en un lenguaje de marcado (como HTML). Con CSS, podemos controlar el color, la fuente, el tamaño, el espaciado, el diseño y muchas otras características de los elementos HTML. Esto nos permite crear diseños personalizados y atractivos.

Git: Control de versiones

Git es un sistema de control de versiones distribuido que te permite rastrear los cambios realizados en tus archivos a lo largo del tiempo. Es una herramienta esencial para cualquier desarrollador, ya que facilita la colaboración en equipo, la gestión de diferentes versiones de un proyecto y la recuperación de cambios anteriores si es necesario.

Ciclo de vida de un cambio en Git

El ciclo de vida de un cambio en Git generalmente sigue estos pasos:

  1. Realizar cambios: Editas tus archivos directamente.
  2. Añadir al índice (staging): Usas el comando `git add ` para marcar los cambios que deseas incluir en la siguiente confirmación.
  3. Confirmar los cambios: Usas el comando `git commit -m "Mensaje descriptivo"` para crear una instantánea de los cambios en el repositorio local.
  4. Enviar los cambios a un repositorio remoto: Usas el comando `git push` para enviar tus confirmaciones a un repositorio remoto como GitHub.

Creación de ramas

Las ramas en Git te permiten trabajar en diferentes versiones de tu proyecto al mismo tiempo. Esto es útil para desarrollar nuevas funcionalidades sin afectar la versión principal. Para crear una nueva rama, usa el comando `git branch `. Para cambiar a una rama existente, usa `git checkout `.

Fusionar ramas

Una vez que hayas terminado de trabajar en una rama, puedes fusionarla con la rama principal (generalmente llamada `main` o `master`). Para fusionar una rama, primero debes cambiar a la rama donde quieres realizar la fusión y luego usar el comando `git merge `. Si hay conflictos durante la fusión, tendrás que resolverlos manualmente.

Otros comandos útiles

  • `git status`: Muestra el estado actual del repositorio.
  • `git log`: Muestra el historial de confirmaciones.
  • `git diff`: Muestra las diferencias entre dos versiones de un archivo.
